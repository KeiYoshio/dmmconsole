# Release Notes – v1.2.0

**Release date:** 2026-02-26

---

## Overview

This release adds a fully functional USB direct driver for ADCMT 7451A
and improves the settings-change behavior during measurement streaming.

---

## What's new

### ADCMT 7451A USB direct mode

The USB direct driver (`ad7451a_usb`) for ADCMT 7451A is now fully functional.

To use USB direct mode:

1. Set the instrument language to **ADC** from the front-panel menu:
   MENU → 8 I/F → LANG → **ADC**
2. Connect the USB cable to the PC.
3. In the DMM panel, select **AD7451A (USB)** and interface **USB direct**.

Key requirements for this mode:

- The instrument enters remote mode only after a USB488 REN_CONTROL request.
  Without it, the device ignores all commands except `*RST`.
- Triggers use USB488 TRIGGER (MsgID=0x80), not the text command `*TRG`.
- After each function, range, or NPLC change, a priming measurement cycle is
  performed automatically to absorb a dead cycle in the instrument firmware.

> Tested on **ADVANTEST AD7451A** (the former brand name; functionally identical
> to ADCMT 7451A).

### Seamless settings changes during streaming

Changing function, range, or NPLC while the waveform is streaming no longer
requires a manual stop/restart.  The stream is stopped automatically before the
setting is applied and restarted immediately after.

When switching measurement functions, the default range is also applied in the
same operation, reducing the number of interruptions to the stream.

## Requirements

- Python 3.10+
- [agilent82357b](https://github.com/KeiYoshio/agilent82357b) v1.0.0 (for GPIB)
- Node.js (for frontend rebuild only)

## How to run

```bash
sudo uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

Then open `http://<server-ip>:8000` in a browser.
