# v1.4.0 – Arbitrary Waveform Upload, Counter UI Enhancements & Waveform Table Fix

## Summary

Adds an **ARB tab** for generating arbitrary waveforms from mathematical formulas, previewing them in real time, and uploading to the FY6800's DDS memory (Arb 1–64). Extends the Counter tab with additional readings and control buttons. Fixes waveform names and CH1/CH2 ID mappings based on [fygen](https://github.com/mattwach/fygen). Adds a frequency unit selector (uHz–MHz) to CH1/CH2 controls.

## New Features

### ARB (Arbitrary Waveform) Tab
- **Formula input**: Define waveforms as `f(x)` where x = 0..1 (8192 points). Math functions (sin, cos, pow, exp, …) and constants (pi, e) are available. Results are automatically normalized to 14-bit (0–16383).
- **Presets**: Sine, Sawtooth, Triangle, Gaussian, Sinc, Staircase, Rectified, Clipped Sin — one-click formula insertion with instant preview.
- **Preview chart**: Chart.js line graph showing the waveform in real time (1024 points, downsampled from 8192). Y-axis fixed at 0–16383.
- **Upload**: Select an Arb slot (1–64) and optionally a target channel (CH1/CH2). On success, the channel's waveform is automatically switched to the uploaded slot.
- **Conflict check**: Upload is blocked if the target slot is currently active on either channel.
- **Help dialog**: Explains formula syntax, loop boundaries, and shared slot memory (CH1/CH2 share the same Arb slots).

### Backend — Waveform Upload
- `FY6800Serial.write_bytes()` — raw binary send without LF termination or delay.
- `FY6800.upload_waveform(slot, samples)` — DDS_WAVE handshake → 14-bit LE split encoding → binary data transfer → HN response polling (up to 15 s).
- `POST /api/upload_waveform` — new REST endpoint (`UploadWaveformRequest`: slot 1–64, samples list of 8192 ints).

### CH1/CH2 Frequency Unit Selector
- Frequency input field now includes a unit selector: uHz / mHz / Hz / kHz / MHz.
- Decimal input supported. Leverages FY6800's full μHz resolution (14-digit encoding).
- Switching units automatically re-displays the current value in the new unit.

### Counter Tab Enhancements
- **Additional readings**: Period, positive pulse width (+Width), negative pulse width (-Width), and duty cycle — displayed in real time with auto-scaling (ns → μs → ms → s).
- **Coupling toggle**: AC (Front Counter) / DC (Rear Trig) via WCC0/WCC1.
- **Reset button**: Counter reset via WCZ0.

### Arbitrary Waveform Upload Protocol (test suite)
- `test_fy6800.py` test 8: upload protocol implementation based on fygen analysis.
- Protocol: `DDS_WAVE{1..64}` → `W` response → 8192 samples × 14-bit LE split (16384 bytes) → `HN` response.
- **Verified on hardware** (2026-03-01): DC flat, Square, and Sine patterns all received `HN` response. Counter verification PASS (DC → 0 Hz, Square/Sine → 10,000 Hz).
- `STOPBITS_ONE` works fine (fygen uses `STOPBITS_TWO` but no difference observed).

## Changes

### Waveform Table (fygen-based)
- Waveform names corrected to match actual FY6800 hardware, based on fygen's `wavedef.py`.
- CH1 and CH2 now use separate waveform ID mappings (CH2 = CH1 − 1 from Adjustable Pulse onward).
- Adjustable Pulse is CH1-only — removed from CH2 waveform dropdown.
- FY6900-only waveforms (Rectangle, Trapezoid) excluded.
- Arbitrary Waveform entries (Arb 1–64) added to waveform list (CH1: ID 34–97, CH2: ID 33–96).

### API
- `CommandRequest` extended with `counter_reset` (bool) and `coupling` (str) fields.
- `GET /api/channel_state/{channel}` now returns per-channel `waveform_options`.
- `POST /api/upload_waveform` added.

## Bug Fixes
- Fixed Adjustable Pulse appearing in CH2 waveform dropdown (CH1-only waveform).
