# Release Notes – v1.1.0

**Release date:** 2026-02-24

---

## Overview

This release renames the application to **Instrument Console** and introduces
two major additions: a home landing page and a raw SCPI terminal.

---

## What's new

### Home page

The app now opens at `/` with a card-based launcher.
Clicking a card navigates to the corresponding tool (`/dmm`, `/terminal`, …).
This makes it easy to add new instrument panels in future releases.

### SCPI Terminal (`/terminal`)

A new raw command console for any connected instrument:

- **Two-pane layout** – sent commands (TX) and received responses (RX) displayed
  in separate scrollable panes with timestamps and sequence numbers.
- **Line-ending control** – CR and LF toggle buttons let you choose what is
  appended to each command (`[CR]`, `[LF]`, `[CRLF]`, or none).
- **Continuous entry** – the command input keeps focus after each send so you
  can type the next command immediately.
- **Independent connection** – the terminal holds its own GPIB/LAN/USB connection,
  separate from the DMM panel.  Both can be connected at the same time.

### USB instrument discovery

The **VISA Resource** field in both the DMM and Terminal connection bars is now
a combobox.  Clicking or focusing the field scans connected USB devices via
`pyusb` and shows a dropdown of candidates:

```
HEWLETT-PACKARD HP 34401A
  USB0::0x1334::0x0202::662500377::INSTR
```

Non-instrument USB devices (keyboards, mice, storage, hubs, audio, video) are
automatically excluded from the list.  You can still type a VISA string manually
if your device is not listed.

## Requirements

- Python 3.10+
- [agilent82357b](https://github.com/KeiYoshio/agilent82357b) v1.0.0 (for GPIB)
- Node.js (for frontend rebuild only)

---

## How to run

```bash
sudo uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

Then open `http://<server-ip>:8000` in a browser.

---

## Known issues

- **ADCMT 7451A USB direct mode** – write commands are silently ignored by the
  instrument.  Root cause not yet identified.  GPIB connection works correctly.
- Tested on macOS only.  Linux expected to work.  Windows untested.
